Задача H. Файлы для Рэдрика

Имя входного файла: input.txt
Имя выходного файла: output.txt
Ограничение по времени: 1.5 секунды
Ограничение по памяти: 256 Mb

В задаче G Рэдрик заказал знакомому программисту сгенерировать шаблоны таблиц длины N. С сожалению, он не учел, что таких шаблонов срезвычайно много. И если записывать из в отдельные файлы, то с такими данными неудобно работать. Поэтому решили записывать шаблоны в файлы подряд в лексикографическом порядке. Но если их записывать в один файл, то получится файл гигантских размеров, с которым тоже неудобно работать. Тогда решили , что можно писать в файлы так, чтобы размер каждого файла не превышал K бвйтов. Если текущий шаблон не влазит в текущий файл, программист создает новый файл.

Рэдрика интересует минимальное количество файлов, в которые можно разместить все шаблоны длины N. А также, каков будет размер последнего файла.

Входные данные

Единственная строка входного файла содержит два целых числа N и K. Значение N не превышает 13. Значение K таково, что любой шаблон длины N помещается в файл размера K.

Выходные данные

В результирующий файл Вы должны вывести два числа - количество файлов, в которые можно записать все шаблоны длины N, а также размер последнего файла.

Пример входного и выходного файлов

input.txt        output.txt        
2 30

2 30
